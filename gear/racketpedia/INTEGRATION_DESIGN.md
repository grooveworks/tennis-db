# Racketpedia × Tennis DB 統合設計（2026-06-26 合意版）

長い議論で確定した土台。実装前の正本。**実装は別の集中作業**（protected app・データ移行・破壊リスク高）。

## 確定した前提（痛い学びの結論）
- **自動マージは不可**（実データ28本で正しく結べたのは1〜2本。誤マッチ多発）。やらない。
- **評価は条件依存**：同じ弦でも(弦＋テンション＋ラケット＋イベント)が違えば別評価。**重複扱いで統一・最新だけ残すは禁止**（本質破壊・ユーザー最重要原則）。
- 価値の天井は低め（ラボ数値は大半が会員ゲートの裏／体感は組合せに紐づき単体データとズレ）。承知の上で、機材タブの評価表の"客観の背骨"として使う範囲。

## データモデル（合意）
- **コンポーネント＝個別ID**：ストリング・ラケット・イベントは各々ID。Tennis DBのstrings/racketsには既にIDがある。**問題=記録が"名前"で参照していること**→これをID参照へ移行するのが本丸。テンションは値(ID不要)。
- **評価＝組み合わせのレコード（複数）**：1評価＝(縦弦ID＋横弦ID＋ラケットID＋テンション＋イベントID)＋体感/評価。**フル組み合わせに新IDは発行しない**（身元は部品ID＋テンションから自動で決まる合成キー。新ID発行は重複組み合わせ＝識別地獄を再発させるので禁止）。**同じ組み合わせでもイベントごとに別レコードで全部残す**（条件評価を潰さない）。
- **名前付き定番setup**（縦×横）は既存の stringSetup のまま（IDあり）。
- **弦の名前＝ラケペディアの正式名を採用**（表記ゆれ解消）。**弦のID＝Tennis DB既存を維持**。注意:記録は名前参照なので、改名するなら**記録内の名前も書き換える移行**が要る（既存「表記ゆれ統合」機能＋バックアップ＋確認）。

## Racketpedia データの置き場（実測根拠）
- あなたの弦(~24本)の**最小データ(名前/太さ/静的剛性/8軸)=約4KB → Tennis DBの中**(軽い・同期・オフライン可)。
- **全カタログ(1104本・全項目=約1.3MB)→ Tennis DB の外**。Firestore 1ドキュメント上限1MBを超えるので入らない。**Firebase Storage**(推奨)または遅延読込の静的ファイル。比較/発掘画面を開いた時だけ読込・端末キャッシュ。core にも Firestore(DB) にも入れない。取り込みパイプラインが更新。

## 根本原因（2026-06-26 ユーザー診断・最重要）
**弦もラケットも、name にブランド名とモデル名(と太さ)が混ざっているのが、全ての散らかりの根本原因**。だから「EZONE 100 TOUR」と「YONEX EZONE 100 TOUR」が割れる/「Babolat ALU Power」のようにブランドを取り違える(ALU Power は Luxilon)/照合が壊れる。→ **brand を独立項目に分離する**のが土台。Racketpedia も brand+model 別々なので、分離すれば brand+model+太さで確実に突合できる(名前スープ照合の1〜2/28から脱却)。

## 実装順（壊さない）
1. **ブランド／モデル／太さの分離（根本治療・土台）**：strings/rackets の name → brand + model(+弦は太さ)に分ける。brand は正規リストに統一。曖昧(ALU Powerのブランド・多語ブランド Pro Kennex/Big Breakers・試打のみのラケット)は**本人確認**。これで表記ゆれ消滅＋Racketpedia突合が成立。
2. **掃除**：condition評価のdedupはしない。genuine error(例: 'active'異常値・EZONE表記ゆれ)だけ本人確認で。決めつけで掃除しない。
3. **ID化**：記録(練習/試合878件・setup・racket)の弦/ラケット参照を name→ID へ移行。**全記録に触れる＝データ破壊地雷**。バックアップ＋元に戻せる＋段階的＋ユーザー確認が絶対。
4. **Racketpedia連携**：あなたの弦~24本を slug に結ぶ(assisted-manual・自動でない・色/太さは本人が選ぶ)。無い弦(ProsPro等)はslug無しで評価のみ。最小データをTennis DBに、全カタログは外。

## データ安全(絶対)
バックアップ→元に戻せる→一気にやらず段階的→各データ変更でユーザー確認。**条件ごとの評価は絶対に潰さない**。
