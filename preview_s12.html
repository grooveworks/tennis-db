preview_s12 — Session 追加 (FAB + QuickAdd 2 択)

S12 のスコープ: FAB タップ → ミニメニュー (大会 / 練習 の 2 択) → QuickAdd モーダル (最小フォーム) → 作成 → SessionDetailView (Detail 閲覧モード)。
試打は v2/v3 で最も作り込まれた機能のため、Home タブ (S14) の 3 ボタン経由で専用設計する方針。FAB ミニメニューには含めない。

S12 の主要決定事項

  • FAB ミニメニュー: 大会 / 練習 の 2 択（試打は除外、S14 Home 3 ボタンに集約）
  • QuickAdd モーダル方式: 最小フォーム (v3 踏襲)。SessionEditView を空起動するフルフォームではなく、出先 30 秒入力を優先
  • 大会 QuickAdd フィールド: 日付* / 大会名* / 形式 / クラス / 会場 / 公開設定
  • 練習 QuickAdd フィールド: 日付* / 種別 / イベント名 / 会場 / 開始-終了時刻 / 公開設定
  • 作成後の遷移: SessionDetailView (閲覧モード)。直後に画面を閉じやすい配置 (アクション完了後の自然な離脱)
  • 必須未入力で「作成」ボタン disabled: SCHEMA REQUIRED_FIELDS 駆動 (S11 と同じ)
  • blank helper: src/domain/blank.js 新規 (3 種別とも作成、blankTrial は S14 で再利用)

② 大会 QuickAdd モーダル (最小フォーム)

ミニメニュー「大会」タップで表示。最小限の 6 フィールド。詳細は作成後の Detail 画面 → 編集ボタンで追記。
必須は 日付 と 大会名。未入力時は「作成」ボタン disabled、フィールド赤枠 (S11 と同じバリデーション)。

大会 QuickAdd (入力中)
大会を追加
まず作成 → 結果 / 機材 / 試合記録は後から編集ボタンで追記。30 秒で記録できます。
大会 QuickAdd (必須未入力で disabled)
大会を追加
必須項目です

③ 練習 QuickAdd モーダル (最小フォーム)

ミニメニュー「練習」タップで表示。最小限の 7 フィールド (大会より時刻が 1 行多い)。
必須は 日付 のみ。種別 / イベント名 / 会場 / 開始-終了時刻 / 公開設定。
v3 と同じく開始-終了入力で時間 (duration) は自動計算。

練習 QuickAdd
練習を追加
機材 / 体調 / メモは後から編集で追記。練習時間は時刻から自動計算 (90 分)。

④ 作成 → SessionDetailView (Detail 閲覧モード)

「作成」ボタンタップで:

  • localStorage + Firestore に保存 (S11 同様の save() 経由)
  • Sessions リスト state に即時反映 (新規行が追加される)
  • QuickAdd モーダルが閉じる
  • SessionDetailView が 閲覧モードで slide-in
作成直後は最小フィールドしか入っていないので、Detail はスカスカ。「もっと書きたい時は編集ボタン」「画面を閉じてリストに戻る」のどちらでも OK。

確認頂きたい点

  • ① FAB ミニメニュー: 大会 / 練習 の 2 択 (試打除外、S14 Home 3 ボタンに集約)
  • ② 大会 QuickAdd: 6 フィールド (日付*/大会名*/形式/クラス/会場/公開設定)。それ以外は編集で追記
  • ③ 練習 QuickAdd: 7 フィールド (日付*/種別/イベント名/会場/開始-終了時刻/公開設定)。時刻入力で duration 自動計算
  • ④ 作成後の遷移: SessionDetailView (閲覧モード) で slide-in。空フィールドは「—」または empty hint で表示
  • 必須バリデーション: 日付/大会名 (大会)、日付 (練習)。未入力で「作成」disabled + 赤枠
  • 会場・公開設定: master データから Select (S11 MasterField を再利用)

OK または修正指示を頂ければ、§1 着手前チェックリストを埋めて実装承認依頼に進みます。
