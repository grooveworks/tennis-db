## 0. 構造共通ルール

全画面に共通する骨格:

```
┌─────────────────────────┐  ← ヘッダ (56px 固定)
│ [←] 画面タイトル   [⋯]  │     左: 戻る / 右: メニュー
├─────────────────────────┤
│                         │
│     メインコンテンツ     │  ← スクロール領域
│     （カード群）         │
│                         │
├─────────────────────────┤
│ 🏠 📋 🎾 📝 📊         │  ← TabBar (56px + safe-area)
└─────────────────────────┘
```

- **左上 = 戻る** (ホーム以外の全画面)
- **右上 = メニュー/設定**
- **下部タブ = 5 タブ**（Home / Sessions / Gear / Plan / Insights）
- 主アクション（+追加 等）はコンテンツ下部に固定配置、もしくはタブバー上の FAB

---

## 1. 画面一覧

| # | 画面名 | 用途 |
|---|---|---|
| 1 | ヘッダ + TabBar | 全画面共通の骨格 |
| 2 | Home タブ | 登録導線（3ボタン）、Next Actions、近日予定 |
| 3 | Sessions 一覧 | 大会/練習/試打を時系列カード表示 |
| 4 | Sessions 展開 | カードタップ時に詳細概要を展開 |
| 5 | Session Detail | 全画面編集フォーム |
| 6 | Modal / ConfirmDialog | 入力モーダル、削除確認 |
| 7 | Toast | 操作フィードバック |

S3 では上記 7 画面をプロトタイプ化。Gear/Plan/Insights/インポート/マージは後 Stage。

## 3. 画面遷移図

```
         [Home]
          │ │ │ ─── 3ボタン ─── [QuickAdd Modal]
          │ │ └─── Next Actions ─── [NextDetail]
          │ └──── MiniCalendar タップ ─── [Session Detail]
          └─── 最近の好成績タップ ─── [Session Detail]

         [Sessions 一覧]
          │ ├─── カードタップ ─── 展開 (同画面)
          │ │                      └─── [編集] ─── [Session Detail]
          │ │                      └─── [🗑] ─── [ConfirmDialog]
          │ └─── [✏️編集] ─── 編集モード (選択→一括削除/マージ)
          │                   └─── [🔗マージ] ─── [MergeModal]
          │
          └─── TabBar ─── 他タブへ

         [Session Detail]
          │ ├─── [←] ─── 一覧に戻る (保存せず)
          │ └─── [💾] ─── 保存して一覧へ
```

## 4. 視覚プロトタイプ (v4/wireframes.html) について

- 上記 7 画面を縦スクロールで並べた HTML
- モバイル幅（375px）のフレームに入れて、PC ブラウザでも確認しやすく
- DESIGN_SYSTEM_v4.md の具体値を CSS 変数で実装
- Material Symbols を CDN で読み込み
- ダミーデータで埋める（架空の大会名・日付）
- **機能は未実装**（クリックしても動かない、見た目のみ）
