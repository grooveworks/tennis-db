実装方針 (Phase 3 SCHEMA / Phase 4 実装で適用)

領域方針備考
並び順データモデル 各 string / racket に order: number フィールド追加 既存データに無い場合、読込時に index で遅延付与 (core/03_storage.js)
Select 表示順 status 優先度 → order ASC の二段階 sort rejected は order あっても末尾に来る。試打カード登録 / 大会・練習編集 / Sessions 全 Select に適用
並び替え UX セクションヘッダー「並べ替え」リンク → モード ON で ▲▼ ボタン → 「完了」で OFF ドラッグ&ドロップは採用しない (モバイル誤操作リスク)
並び替え永続化 ▲▼ タップで即 Firestore write (debounce bypass、S15.5.3 と同パターン) 「完了」ボタンは UI モード切替のみ (データはタップ毎に保存済)
ラケット側にも適用 同じ仕組みを Racket Board にも (Manage Masters の引退ラケット欄ではない、Racket Board そのもの) Phase 4 で RacketBoard.jsx 実装時に組み込む
tap 領域 ▲▼ ボタンは縦並び 32×22 = 32×46 の塊で、左右の余白も含めると 44px 確保 必要なら 32×24 に拡大検討
Header 修正 (S15.5.7 反映) 全画面 Header から version 削除、⚙️ 設定アイコンを weather と user-circle の間に追加 preview_s16_p1/p2 の Header 表示は実装フェーズで p3 に合わせる
auto-save (S15.5.9 標準方針の継承) Racket Edit Modal にも auto-save 実装、draft key = yuke-racket-draft-{id}-v1 Decision Notes / note 系 textarea は auto-grow + --memo-font-scale 適用
完了テスト規律 (S15.5.8 教訓) S16 完了時は「ラケット追加 → 編集 → 保存 → 並べ替え → 試打追加 → 削除」サイクルを必ず通す S15.5.8 の MENTAL_LABELS 定義漏れ事故の再発防止
