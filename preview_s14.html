② 試打 QuickAdd モーダル — 3 ボタン「試打を記録」タップ後

QuickAddModal を type="trial" 拡張。最小フォーム (日付 / ラケット* / 縦糸+横糸 / テンション縦+横 / 判定 3 大ボタン / 会場) のみ。詳細評価 (打感 6 + 特性 2 + ショット 8 + 安心感 1) と連携先は 作成後 Detail → 編集ボタンで追記。

  • 必須: 日付 / ラケット (SCHEMA required:true 駆動、S11 と同じ getRequiredErrors)

  • visibility 公開設定は trial には無い (DECISIONS S11)、その分シンプル
  • 保存後: SessionDetailView (mode="detail") で slide-in、空項目は「—」表示。S11 で確立済の遷移を再利用

このプロトで確認してほしいこと

1. 3 ボタン色分け: 大会 Orange / 練習 Green / 試打 Purple で十分判別できますか? 横並び 3 列 (64px 高) で親指届きますか?

2. MiniCalendar の濃淡: 練習 3 段階 (薄緑 / 中緑 / 濃緑) と大会 (淡 Orange + trophy アイコン) の重なりは読み取れますか? Phone A 4/11 マスは「大会 + 試打 (紫ドット)」、4/10 マスは「練習 60 分超で濃緑+白文字」を表現しています。

3. インライン DayPanel: Phone B のように MiniCalendar カード内で展開する形式 (vs CalendarView §8.5.7.1 の本格 DayPanel) で問題ないですか? ホームは多数セクションが縦に並ぶため、外周モーダル展開だと文脈が断絶すると判断しました。

4. 試打 QuickAdd: 「日付 / ラケット* / 縦糸+横糸 / テンション縦+横 / 判定 3 ボタン / 会場」の 7 項目で出先入力に必要十分ですか? 評価 17 項目は Detail → 編集で OK ですか?

5. Next Actions の「全て見る」: タップで Plan タブ (S17 まで placeholder) に切替で良いですか? 「読み取りのみ」を明示する補助文 (e.g. 「Plan タブで管理」) を入れた方が良いですか?

6. 好成績の上限: 5 件 (優勝 → 準優勝 → 3 位 → 予選突破の優先順、同優先内は日付降順) で良いですか? それとも 3 件で十分ですか?

7. 順番: 「3 ボタン → Next Actions → MiniCalendar → 近日予定 → 好成績 → Stats」の縦並び順で良いですか? (v3 と同じ並び。Next Actions は機能未実装な分上に出すべきか / 下げるべきか)
