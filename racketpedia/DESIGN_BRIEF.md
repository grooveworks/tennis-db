# ストリング比較UI — デザイン依頼 (claude.ai / Artifact 用)

このファイル1枚を claude.ai に渡せば、文脈ゼロから設計を始められます。
**見た目の設計を claude.ai が担当 → 完成した Artifact(コード)を Claude Code が Tennis DB に実装**、という分担。

---

## 依頼

テニスのストリング(弦)比較UIを **Artifact (React か HTML)** で作ってください。**PC/タブレット向け**。
**見やすさ最優先・奇抜さ不要。発明しない。** 既存の定番の見せ方を踏襲してください。余白を活かし、密集させない(密な文字/図の塊が苦手)。

### 踏襲する定番(リサーチ済み)
- **Racketpedia 本家**: 1行 = 弦名 + 素材 + **強みタグ3つ**(数値の羅列でなく「何が強いか」をタグで)。人気/最近分析などでセクション分け。
- **Tennis Warehouse University(TWU)**: 最小の一覧を、**選んだ1指標(パワー/スピン/コントロール/腕に優しい/素材/ゲージ)で並べ替え**、行クリックで詳細展開。

## 目的
自分が試した弦・候補の弦を、**客観データ(ラボ計測)** と **自分の試打体感** を重ねて比較し、ラケットに合う弦を選ぶ。

## 1弦あたりのデータ
- 名前 / ブランド / ゲージ(mm)
- 構造(モノ/マルチ/ガット) / 素材(コポリエステル/コポリアミド/ナイロン/ナチュラルガット 等) / 断面形状(ラウンド/八角/六角/五角/ねじれ/粗面 等)
- 静的剛性(kg/mm) / 動的剛性(g/mm) ※剛性は物理値=1つ
- 8軸スコア 0-100: **パワー / 弾き / 弾性 / スピン / コントロール / テンション維持 / 安定性 / 快適性**
- 自分の試打体感: 上の8軸の主観スコア(記録した弦のみ・任意)
- 自分の評価: 本命/可/微妙/却下/未試打 等の一言

## 核(最重要)
**客観(ラボ)と主観(試打)を同じ8軸レーダーに重ねる。ズレが情報**(例: あるコントロール軸が客観100でも体感45なら、その差が選定のヒント)。
一覧でもスコア列に **客観/主観を併記**し、ズレが大きい所を強調。色は **客観=青 / 主観=橙**。

## 操作
- 名前検索
- 絞り込み(ドロップダウン): 素材 / 形状 / フレーム
- 並べ替え: 剛性 / パワー / スピン / コントロール / 快適 / 維持

## フレーム適合
フレームを選ぶと、合う剛性帯の弦に「推奨」を出す。**低パワーのフレーム→柔らかめ / 高パワー→硬め(逆相関)**。
(例: BOOM PRO=低パワー→柔らかめ推奨 / EZONE=高パワー→硬め推奨)

## 詳細(行クリック)
8軸レーダー(客観+主観の重ね) + 試打メモ(本人の言葉) + 剛性の裏付け。

## データの注意
- 動的剛性・帯別剛性・8軸レーダーは **有料会員のラボデータ**。未取得は「空欄」で表示(推測で埋めない)。
- 実データは全1,104弦(`racketpedia/out/strings.json`)。下はデザイン用の代表サンプル。
