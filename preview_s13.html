preview_s13 — Session 削除 + cascade ConfirmDialog

Stage S13 = Session 削除時に linkedPracticeId / linkedMatchId を持つ試打 (trial) の連携を自動クリアする安全削除。
確認ダイアログには影響件数を事前提示し、試打自体は残す(連携 ID のみ空文字に書き換え)。
本プロトは ConfirmDialog 文言とトーストの 5 ケースを視覚確認するためのもの。実装前のレビュー用。

cascade 仕様まとめ

削除対象cascade 対象クリアする field影響件数の算出
大会 (tournament)その大会の matches[].id を linkedMatchId に持つ triallinkedMatchId = ""matches[] を走査して trial を逆引き
練習 (practice)linkedPracticeId === practice.id の triallinkedPracticeId = ""trial 1 走で抽出
試合 (match) ※大会編集画面linkedMatchId === match.id の triallinkedMatchId = ""trial 1 走で抽出
試打 (trial)(無し — 試打は連携元、cascade 対象外)—件数表示しない

レビューしてほしい点

  • Case B/C/D の本文 (「3 件 の連携情報が外れます (試打自体は残ります)」) で意味が伝わるか
  • 「連携情報」という言葉が不自然なら別の言い回しに変える (例: 「連携が外れます」「紐付けが外れます」「リンクが外れます」)
  • Case E の toast 2 行構成 (1 行目=削除完了, 2 行目=cascade 件数 の薄字) で十分か、別 UI が要るか
  • Case D の試合行に「(試打 1 件 連携中)」のヒント表示を 常時付けるかどうか (削除前にユーザーが気付ける)
