## 1. 1 ゲームのスコア

| スコア | 呼び方 |
|---|---|
| 0 | ラブ (love) |
| 1 ポイント目 | 15 |
| 2 ポイント目 | 30 |
| 3 ポイント目 | 40 |
| 4 ポイント目 | ゲーム |

3-3（40-40）= **デュース**。デュース後は 2 ポイント連続取得でゲーム勝利。
- アドバンテージ（先に 1 ポイント取った側） → もう 1 ポイントで勝ち
- 取られたら → デュースに戻る

**ノーアド方式**（一部の草トー / ジュニア）: デュース時に次の 1 ポイントでゲーム決着。レシーバーがサイド選択。

## 2. コートチェンジ（チェンジオーバー）

**ITF Rule 11 — End change**:
> The players shall change ends at the end of the first, third and every subsequent alternate game of each set, and at the end of each set unless the total number of games in such set is even, in which case the change is not made until the end of the first game of the next set.

### 2.1 通常時 — 奇数ゲーム終了後にチェンジ

- ゲーム 1 終了後 → ✓ チェンジ
- ゲーム 2 終了後 → なし
- ゲーム 3 終了後 → ✓ チェンジ
- ゲーム 4 終了後 → なし
- ゲーム 5 終了後 → ✓ チェンジ
- ゲーム 6 終了後 → なし
- 以下同様

判定式: `games.length % 2 === 1` （奇数）

### 2.2 セット終了時の特例
- セットの合計ゲーム数が**奇数**（例 6-3 = 9） → ✓ セット終了時にチェンジ（同時にセット間休憩）
- セットの合計ゲーム数が**偶数**（例 6-4 = 10） → セット終了時はチェンジせず、次セットのゲーム 1 終了後にチェンジを持ち越し

**Tennis DB の扱い**: setScores は文字列配列（`["6-3", "6-4"]` など）でセット境界をゲーム単位で追跡しない。GameTracker の games[] は単一の試合内の連続ゲーム列。よってセット境界の特例は実装に含めない（奇数判定のみ）。

### 2.3 休憩時間
- 通常チェンジ: 90 秒（ITF 公式）
- ゲーム 1 終了後だけ: 一部の運用では休憩なし（即座にチェンジのみ）— Tennis DB は区別しない

### 2.4 タイブレーク中のチェンジ
- 6 ポイントごとにチェンジ（合計 6, 12, 18 ポイント目）
- タイブレーク中は休憩なし（座らない）

**Tennis DB の扱い**: タイブレークは setScores に "7-6(5)" のように記録するが、ポイント単位は DataTennis アプリ範囲。GameTracker はゲーム単位なので、タイブレークは「1 ゲーム」として扱う。

## 3. セット

| 形式 | 内容 |
|---|---|
| 標準 | 6 ゲーム先取（差 2 以上）、6-6 でタイブレーク |
| アドバンテージセット | 6-6 後もタイブレークなしで差 2 まで継続（グランドスラムは廃止） |
| プロセット | 8 ゲーム先取（草トーで使用） |
| 4 ゲームセット | 4 ゲーム先取（短時間運用） |
| スーパータイブレーク | 第 3 セットの代わりに 10 ポイント先取（差 2） |

### 3.1 草トー / Tennis DB で扱うセット形式 (S18 リクエスト 30-e で対応)

`tournament.matchFormat` / `match.format` でセット取得条件を制御する 4 プリセット + カスタム:

| プリセット (`preset`) | 通称 | 1 セット取得条件 | TB 発動 | TB ポイント | 試合決着 |
|---|---|---|---|---|---|
| `1set` | 1 セットマッチ (= 6 ゲームマッチ) | 6 ゲーム先取・**差 2 必要**、5-5→7-5 | 6-6 | 7 | 1 セット先取 |
| `3set` | 3 セットマッチ | 同上 | 6-6 | 7 | 2 セット先取。**`finalSetMode:"matchTiebreak10"` トグルで 1-1 後 10pt TB 置換** |
| `6game` | 6 ゲーム先取 (草トー時短) | 6 ゲーム取った時点・**差 1 OK**、5-5→6-5 | なし | — | 6 ゲーム到達 |
| `4game` | 4 ゲーム先取 (ショートセット) | 4 ゲーム取った時点・**差 1 OK**、3-3→4-3 | なし | — | 4 ゲーム到達 |
| `custom` | カスタム | 上記 6 パラメータ全部手動指定 | (任意) | (任意) | (任意) |

**重要な区別**:
- 「6 ゲームマッチ」(= 1 セットマッチ) と「6 ゲーム先取」は別物。前者は差 2 必要 + 6-6 で TB、後者は差 1 で OK + TB なし。
- 「マッチタイブレーク」= 3 セットマッチで 1-1 になった時、第 3 セットの代わりに行う 10 ポイント TB (= スーパータイブレーク、全豪 2019- 等)。Tennis DB は `finalSetMode:"matchTiebreak10"` で表現。
- `setMinDiff:1` のとき (6/4 ゲーム先取の草トーモード) は `tiebreakAt:"never"` を採用 (草トーで 5-5 TB はあるが稀、必要ならカスタム使用)。

## 4. タイブレーク

- 6-6 で発動（標準）
- **7 ポイント先取、差 2 以上**
- 最初のサーバーが 1 ポイント、以降 2 ポイントずつ交代
- 6 ポイントごとにコートチェンジ
- 表記: 7-6(5) = タイブレーク 7-5 で取った
- 10 ポイント版（スーパータイブレーク、マッチタイブレーク）: 第 3 セット代用 / ダブルス短縮

---

## 5. マッチ形式

| 形式 | 内容 | 主な使用場面 |
|---|---|---|
| 3 セットマッチ (best of 3) | 2 セット先取 | 草トー / 市民大会 / WTA / ATP 通常 |
| 5 セットマッチ (best of 5) | 3 セット先取 | グランドスラム男子 / デ杯 |
| 1 セット + スーパータイブレーク | 1 セット取った後、勝負がつかなければ 10 ポイント先取 | 短時間運用 / ダブルス |
| プロセット | 1 セット 8 ゲーム先取 | 草トー / 練習試合 |

---

## 6. サーブ順

### 6.1 シングルス
- 各ゲームでサーバーが交代（同一選手が連続でサーブしない）
- セット内で固定パターン（ゲーム 1: A, ゲーム 2: B, ゲーム 3: A, …）

### 6.2 ダブルス
- 各サイドはセット開始時にサーブ順を決定
- セット内でサーブ順は固定（変更不可）
- 4 人で順番に: A1 → B1 → A2 → B2 → A1 → … と巡回
- セット終了時にチームで順番変更可

### 6.3 タイブレーク
- 最初の 1 ポイントは 6-6 終了後にレシーブ側だった選手がサーブ
- 以降 2 ポイントずつ交代

---

## 7. ダブルス特有

- レシーブサイド（デュースサイド / アドサイド）はセット開始時に決定、セット内で固定
- ペアの 1 人がデュースサイド、もう 1 人がアドサイド
- ボレーポーチ・並行陣・雁行陣などフォーメーションはルール外（戦術）

---

## 8. ミックスダブルス

- 男女ペア
- サーブ順 / レシーブサイドはペアで自由に決定
- 男性が女性にサーブする / 女性が男性にサーブする どちらも可

## 9. 試合形式と Tennis DB データの対応

| Tennis DB フィールド | 値の例 | 対応するルール |
|---|---|---|
| `tournament.type` | "singles" / "doubles" / "mixed" | §6 サーブ順 / §7-8 ダブルス・ミックス |
| `tournament.matchFormat` | `{preset:"3set", setTargetGames:6, ...}` | §3.1 プリセット + パラメータ |
| `match.format` | `null` (大会継承) または同形式 object (上書き) | §3.1 試合ごとの形式上書き |
| `match.setScores[]` | `["6-3", "7-6(5)"]` または `["6-3", "4-6", "10-7"]` | §3 セット / §4 TB / §3.1 マッチ TB は 3 セット目位置 |
| `match.games[]` | `[{winner:"me"}, {winner:"opp"}, ...]` | §2 ゲーム単位 |
| `match.changeovers[]` | `[{afterGame:1, mental, physical, note}, ...]` | §2.1 奇数ゲーム後の CO 記録 |
| `match.result` | "勝利" / "敗北" / "棄権" / "" (未定) | §5 マッチ形式の勝敗判定（手動入力 / format 自動） |

### 9.1 format object の構造 (§3.1 参照)
```js
{
  preset: "1set" | "3set" | "6game" | "4game" | "custom",
  setTargetGames: 4 | 6 | 8,    // セット取得ゲーム数
  setMinDiff:     1 | 2,         // 必要差
  tiebreakAt:     "never" | "5-5" | "6-6" | "4-4",
  tiebreakPoints: 7 | 10 | null,
  bestOfSets:     1 | 2,         // 1=1セット先取, 2=2セット先取(=3セットマッチ)
  finalSetMode:   "normal" | "matchTiebreak10",
  noAd:           false,         // ノーアド (今回は data 持つだけ、表示影響なし)
}
```

## 13. Tennis DB 内の CO 判定実装ガイド

GameTracker の CO 入力プロンプト発動条件:

```js
// ゲーム追加直後
if (newGames.length % 2 === 1) {
  // 奇数ゲーム終了後 → CO プロンプト発動
  setPendingCO({ afterGame: newGames.length, mental: 3, physical: 3, note: "" });
}
```
